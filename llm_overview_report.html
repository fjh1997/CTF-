一、什么是大语言模型？

大语言模型（Large Language Model, LLM）是一种基于 深度学习 和 Transformer 架构 构建的 AI 系统，通过海量文本数据训练，能够：

能力 说明
🗣️ 自然语言理解 理解人类语言的含义、语法和语境
✍️ 文本生成 写作文、写代码、翻译、总结
🧠 推理分析 逻辑推理、数学计算、代码调试
🖼️ 多模态处理 理解图片、视频、音频（部分模型）
🔧 工具调用 调用外部工具完成复杂任务
✅技巧

可以把大模型想象成一位通读了互联网大部分文本资料的超级助手，它通过概率预测"下一个词"来生成回答。

四、Token 是什么？

🔤 Token 的基本概念

Token 是大模型处理文本的最小单位。大模型不是直接读"字"或"词"，而是将文本切分成 Token 后进行处理。

举例（英文）：
"Hello, world!" → ["Hello", ",", " world", "!"]  = 4 个 Token

举例（中文）：
"人工智能改变教育" → ["人工", "智能", "改变", "教育"]  ≈ 4~8 个 Token
💡提示

经验法则： - 英文：1 个 Token ≈ 4 个字符 ≈ 0.75 个单词 - 中文：1 个汉字 ≈ 1~2 个 Token - 1000 Token ≈ 750 个英文单词 ≈ 500~700 个汉字

🧩 Token 的种类

类型 说明 举例
子词 Token 最常用，将词拆分为有意义的子片段 "unhappiness" → ["un", "happi", "ness"]
词级 Token 整个单词作为一个 Token "hello" → ["hello"]
字符级 Token 每个字符是一个 Token "hi" → ["h", "i"]
字节级 Token 每个字节是一个 Token，支持所有语言 GPT 系列使用此方式
特殊 Token 模型内部标记 <PAD> 填充、<EOS> 结束

📊 Token 的三大作用

graph LR
    A[Token] --> B[💰 计费单位]
    A --> C[📏 上下文窗口]
    A --> D[⚡ 性能影响]
    B --> B1[输入 Token + 输出 Token = 总费用]
    C --> C1[GPT-5: 128K Token
Claude: 200K Token
Gemini: 2M Token]
    D --> D1[Token 越多
处理越慢
成本越高]

五、如何接入大模型 API

🔑 什么是 API？

API（Application Programming Interface，应用程序编程接口）就像一个 "服务窗口"——你把请求递进去，模型把结果递出来。

graph LR
    A[你的应用/代码] -->|发送请求| B[API 接口]
    B -->|转发请求| C[大模型服务器]
    C -->|返回结果| B
    B -->|返回结果| A

📋 接入步骤（以 OpenAI 为例）

第 1 步：注册并获取 API Key

1. 访问 https://platform.openai.com/
2. 注册账号并登录
3. 进入 API Keys 页面
4. 点击 "Create new secret key"
5. 复制并安全保存你的 API Key（sk-xxxx...）
🔴注意

API Key 就像密码，绝不能泄露！ 不要把它写在公开代码里或分享给他人。

第 2 步：发送第一个 API 请求

Python 代码示例：

# 安装: pip install openai
from openai import OpenAI

client = OpenAI(api_key="sk-你的密钥")

response = client.chat.completions.create(
    model="gpt-5.2",           # 选择模型
    messages=[
        {"role": "system", "content": "你是一位教育专家"},
        {"role": "user", "content": "请帮我设计一个小学数学教案"}
    ],
    max_tokens=2000,            # 最大输出 Token 数
    temperature=0.7             # 创造性程度（0=保守，1=创新）
)

print(response.choices[0].message.content)

第 3 步：理解关键参数

参数 含义 建议值
model 选择使用的模型 根据需求选择
messages 对话消息列表 system + user 角色
max_tokens 限制输出长度 500~4000
temperature 控制随机性/创造性 教案：0.7，代码：0.2
top_p 控制输出多样性 通常 0.9~1.0

八、MCP 协议与工具集成（以 IDA Pro 逆向为例）

🔌 什么是 MCP？

MCP（Model Context Protocol，模型上下文协议）是 Anthropic 于 2024 年推出的开放标准，目标是让 AI 能够连接任何外部工具和数据源。

可以把 MCP 理解为 “AI 的 USB-C 接口”——就像 USB-C 让任何设备都能通过统一接口传输数据，MCP 让任何 AI 模型都能通过统一协议调用外部工具。

graph LR
    A[AI 客户端] <-->|“MCP 协议”| B[MCP 服务器]
    B <--> C[📂 文件系统]
    B <--> D[🗄️ 数据库]
    B <--> E[🔧 IDA Pro]
    B <--> F[🌐 网页浏览器]
    B <--> G[📧 Slack / 飞书]
    
    style A fill:#e3f2fd
    style B fill:#fff3e0

🛠️ MCP 的核心架构

组件 角色 举例
MCP Host（宿主） 运行 AI 的应用程序 Claude Code、Cursor、VS Code
MCP Client（客户端） 在 Host 内负责通信 内置于 Host，无需配置
MCP Server（服务器） 提供工具和数据的服务 IDA Pro MCP、数据库 MCP、文件系统 MCP

📦 常见 MCP 服务器类型

MCP Server 功能 适用场景
Filesystem MCP 读写文件、目录查看 文档处理、代码分析
Database MCP 查询数据库、获取 Schema 数据分析、报表生成
Browser MCP (Playwright) 操控浏览器、截图、测试 网页自动化、UI 测试
IDA Pro MCP 反编译、反汇编、交叉引用 ⭐ 逆向工程自动化
JADX MCP Android APK 反编译 Android 应用分析
Slack / Notion MCP 读取团队消息和文档 团队协作

🔍 实战：IDA Pro MCP 搭建自动化逆向小能手

什么是 IDA Pro MCP？

IDA Pro 是世界最知名的逆向工程工具，通过 MCP 协议接入后，AI 可以直接：

AI 能力 说明
📝 自动反编译 将二进制函数转为可读伪代码
🏷️ 自动命名 给函数/变量取有意义的名字
🔗 交叉引用分析 追踪函数调用关系
📊 结构体识别 自动识别和创建数据结构
💬 添加注释 为代码添加解释性注释
🔍 搜索字符串/字节 在二进制中搜索特定模式

安装与配置步骤

第 1 步：安装 IDA Pro MCP 服务器

# 方式一：PIP 安装
pip install ida-pro-mcp

# 方式二：uvx 直接运行（无需安装）
uvx mcp-server-ida

将 IDA 插件文件复制到 IDA 插件目录：

Windows: %APPDATA%\Hex-Rays\IDA Pro\plugins\
macOS:   ~/Library/Application Support/Hex-Rays/IDA Pro/plugins/

第 2 步：在 IDA Pro 中启动 MCP 服务器

1. 打开 IDA Pro，加载要分析的二进制文件
2. 等待自动分析完成
3. Edit → Plugins → IDA-MCP
4. 控制台显示 "MCP Server started on port 7000"

第 3 步：配置 AI 客户端连接 MCP

在 Claude Code 或 Cursor 的 MCP 配置文件中添加：

// .vscode/mcp.json 或 Claude Code 配置
{
  "servers": {
    "ida-pro-mcp": {
      "command": "uvx",
      "args": ["mcp-server-ida"]
    }
  }
}

或在 Cursor 中：

Settings → MCP Servers → Add Server
→ Name: ida-pro-mcp
→ Command: uvx mcp-server-ida

使用示例：AI 自动分析二进制

配置完成后，你可以直接用自然语言让 AI 做逆向分析：

> 请分析 main 函数的逻辑，反编译并解释每个步骤

> 查找所有引用了 "password" 字符串的函数，分析密码验证逻辑

> 把这个函数的所有变量和函数重命名为有意义的名字

> 分析这个程序的调用图，找出核心处理流程

AI 会通过 MCP 协议自动调用 IDA Pro 的各项功能（反编译、交叉引用、重命名等），然后用自然语言给你解释结果。

✅技巧

MCP 的强大之处在于——同一个 AI 助手可以同时接入多个 MCP Server。例如同时连接 IDA Pro + 数据库 + 浏览器，让 AI 在逆向分析的同时查询漏洞库和在线文档。

第二部分：安装 AI 编程助手 (Claude Code)

环境安装

第 1 步：安装 Node.js

访问 https://nodejs.org/ 下载并安装 LTS 版本（v18 或更高）

安装完成后验证：

node --version    # 应显示 v18.x 或更高
npm --version     # 应显示 9.x 或更高

第 2 步：安装 Claude Code

打开终端（Windows: PowerShell，Mac: Terminal），执行：

npm install -g @anthropic-ai/claude-code

验证安装：

claude --version
claude doctor      # 检查环境是否正常

第 3 步：配置 API Key

在项目目录创建 .env 文件：

# 在项目文件夹中执行
echo ANTHROPIC_API_KEY=sk-ant-你的密钥 > .env

或设置环境变量（推荐）：

# Windows PowerShell
$env:ANTHROPIC_API_KEY = "sk-ant-你的密钥"

# Mac/Linux
export ANTHROPIC_API_KEY="sk-ant-你的密钥"

替代方案：使用 Cursor（图形界面）

如果不习惯命令行，可以选择 Cursor：

1. 访问 https://cursor.com/ 下载安装
2. 打开 Cursor → Settings → Models
3. 添加 Anthropic API Key
4. 选择 Claude Opus 4.6 作为默认模型
5. 在编辑器中按 Ctrl+K 即可 AI 辅助编程

步骤 3：Claude Code 生成的核心代码（示例）

# quiz_generator.py —— Claude Code 自动生成
from anthropic import Anthropic

client = Anthropic()  # 自动读取环境变量中的 API Key

def generate_quiz(subject: str, grade: str, topic: str) -> dict:
    """根据学科、年级和知识点生成测验题目"""

    prompt = f"""你是一位经验丰富的{subject}教师，请为{grade}学生
    针对「{topic}」知识点生成一套测验：

    要求：
    - 5道选择题（4个选项，标注正确答案）
    - 3道填空题（给出答案）
    - 2道简答题（给出参考答案）

    请用 JSON 格式返回。"""

    response = client.messages.create(
        model="claude-opus-4-6-20260301",  # 使用 Opus 4.6
        max_tokens=4000,
        messages=[{"role": "user", "content": prompt}]
    )

    return parse_quiz_response(response.content[0].text)

# 使用示例
quiz = generate_quiz("数学", "初二", "一次函数")
print(f"已生成 {len(quiz['选择题'])} 道选择题")
